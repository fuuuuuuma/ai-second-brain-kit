# corrections.md — 修正指示の蓄積（恒久ルール）

> オーナーから受けた「こう直して」「こうして」という指示を、ここに**1件3行**で蓄積します。
> AIはセッション開始時に必ずこのファイルを読み、ここに書かれた指示を**恒久的に守ります**。
> 同じことを二度言わせないための場所です。

## 書き方（フォーマット）

```
## YYYY-MM-DD
- 指摘：（オーナーが何を直してほしいと言ったか。具体的に）
- 今後：（次から自分がどう動くか。具体的な「する/しない」で）
```

## 運用ルール（AI向け）

- オーナーに訂正・要望を言われたら、**その場で**ここに追記する（後回しにしない）。
- 抽象的に書かない。「丁寧に」ではなく「見出しは体言止めにする」のように具体的な行動で書く。
- 既存の指示と矛盾する新しい指示が来たら、古い行に `（YYYY-MM-DD 更新）` を付けて最新を優先する。
- 月に一度、重複・解決済みを整理する。
