# lint.md — 週次ヘルスチェック

> Vault は足すだけだと、時間とともに矛盾・重複・古い情報が溜まります。
> 「複利で賢くなる」のは、追加と整理を**両輪**で回すからです。週に一度、この観点で点検します。

## いつやるか

- 週に1回（曜日を決める）。または `raw/` を大量に投入した直後。

## チェック観点（AIへの指示）

`wiki/` 全体を読み、次の5点を洗い出して `wiki/lint-issues.md` に列挙する。

1. **矛盾** — 同じ概念が、別ページで違う定義・違う数値になっていないか
2. **欠落** — 3つ以上のノートで言及されるのに、専用ページがない概念はないか
3. **重複** — 似た内容のページが複数ないか（統合候補）
4. **古い情報** — 出典（`raw/`）が消えている／数値の根拠が古いものはないか
5. **新規候補** — 既存ページから派生して1ページ立てる価値のあるテーマはないか

## 出力フォーマット（`wiki/lint-issues.md`）

```markdown
# Lint Issues (YYYY-MM-DD)

## 矛盾
- [ ] 説明（関連: [[ページA]], [[ページB]]）

## 欠落
- [ ] 概念名：N件で言及、専用ページなし

## 重複
- [ ] [[ページX]] と [[ページY]] の統合検討

## 古い情報
- [ ] [[ページZ]] の数値が古い／出典消失

## 新規候補
- [ ] テーマ名：派生理由
```

## 大事なこと

- 自動で勝手に消さない。**洗い出して提案**し、修正はオーナーの承認後に行う。
- 「保存した答えに誤りがあると、その誤りも複利で溜まる」。だから整える工程を飛ばさない。
- 優先度の高い3件をオーナーに提示し、そこから片付ける。
