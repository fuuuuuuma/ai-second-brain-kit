# naming.md — 命名規則と「AIが引き出すための4つの手がかり」

> 「しまう」ためではなく「後で引き出す」ために設計します。
> 将来ノートを探すとき、人もAIも必ず次の4つのどれかを手がかりにします。

## AIが引き出すための4つの手がかり

| 手がかり | 何か | 例 |
|---|---|---|
| **type（種類）** | 何のノートか | 会議録 / 提案 / 調査 / 意思決定 / 顧客メモ / 概念 |
| **when（時期）** | いつのものか | 作成日・関連する時期 |
| **topic（主題）** | 何についてか | 案件名 / 取引先 / テーマ |
| **status（状態）** | どの状態か | 進行中 / 完了 / 保留 / 参照用 |

この4つがノートに記録されていれば、「進行中の(status) 新規事業の(topic) 先月以降の(when) 意思決定(type)を出して」がそのまま通ります。

## ファイル名の型

```
YYYY-MM-DD-種類-主題.md
```

例：
```
2026-05-20-会議録-新規事業レビュー.md
2026-05-18-提案-A社向け改善.md
2026-05-15-調査-競合ベンチマーク.md
```

日付を先頭に置くと、自動で時系列に並び、名前も衝突しません。

## ノート冒頭の frontmatter

各ノートの先頭に、AIが構造として読めるメタ情報を持たせます。

```markdown
---
type: 会議録
status: 進行中
date: 2026-05-20
topic: 新規事業
tags: [新規事業, A社]
---
（ここから本文）
```

`status` が引き出しで最も効きます（「進行中だけ」「完了を除く」が一発）。

## タグの付け方（増やしすぎない）

- 種類で分ける：`#topic/新規事業` `#status/進行中` `#project/A社`
- `/` で区切ると階層として畳めます。
- **5つ以上のノートで使うものだけ** タグ化する（1〜2件のタグはノイズ）。

## 原則

- フォルダで細かく分類しない。命名と `[[リンク]]` で繋ぐ。
- 既存ノートを一括で直そうとしない。**次に作る1枚からこの型で**。過去分は触れたついでに直す。
